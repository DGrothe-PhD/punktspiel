Minimale Lösung zur Übersetzung von App-UI

Startpunkt: Einfache App

Vorbereitete Schritte: App strings von der lokalen Benutzeroberfläche (etwa „Häufige Fragen“, „Einstellungen“, ggf. generell offline Verfügbares) sind bereits in arb-Files untergebracht.

User soll die App in seiner Sprache sehen, falls verfügbar.

Optional: App-Inhalte auf der Webseite dort lokalisiert verwalten.

Code – Minimale Lösung

Wie wäre eine minimale Lösung.

Angenommen, eine App hätte von vornherein nur

class MyApp extends StatelessWidget {

  const MyApp({super.key});

  //final TableExampleApp punkteTabelle = const TableExampleApp();

 

  // This widget is the root of your application.

  @override

  Widget build(BuildContext context) {

    return MaterialApp(

      //locale: appLocale,

      supportedLocales: S.delegate.supportedLocales,

      debugShowCheckedModeBanner: false,

      home: const MyHomePage(title: 'Testapp'),

      localizationsDelegates: const [

        S.delegate,

        GlobalMaterialLocalizations.delegate,

        GlobalWidgetsLocalizations.delegate,

        GlobalCupertinoLocalizations.delegate,

      ],

    );

  }

}

 

Weiter unten im Code tauchen dann Widgets auf wie

Column(children: [

Text(S.of(context).theWeekday),

Text(S.of(context).someOtherText),

...

]

Angezeigt: "Montag", "Was möchten Sie fragen?"

Shared Preferences speichert nur Zahlen, keine Sprache.

 

Wer oder was wählt dann die Sprache aus?

Sehr gut, jetzt sind wir auf der richtigen Flughöhe 👍
Du hast völlig recht: global verteilte Listenables wären Murks – und das brauchen wir alles nicht.

Du stellst jetzt die richtige Grundfrage:

Wenn ich in Flutter locale: NICHT setze – wer entscheidet dann die Sprache?

Die Antwort ist überraschend simpel und genau das ist die minimale, saubere Lösung.

✅ Kurzantwort (die Essenz)

Flutter selbst wählt die Sprache automatisch – anhand der System­sprache des Geräts.

Du musst nichts speichern, nichts verwalten, keine globale Variable haben.

🔍Was passiert konkret in DEINEM Minimal-Setup?

Du hast:

MaterialApp(

  supportedLocales: S.delegate.supportedLocales,

  localizationsDelegates: const [

    S.delegate,

    GlobalMaterialLocalizations.delegate,

    GlobalWidgetsLocalizations.delegate,

    GlobalCupertinoLocalizations.delegate,

  ],

)

Und im UI:

Text(S.of(context).theWeekday)

Text(S.of(context).someOtherText)

Ergebnis:
  • Gerätssprache = Deutsch
    → Flutter wählt Locale('de')
    → S.of(context) liefert Deutsch
  • Gerätssprache = Englisch
    → Locale('en')
    → Englischer Text

👉 Ganz ohne dein Zutun

🧠 Wer genau trifft die Entscheidung?

Flutter macht intern:

  1. OS meldet z. B.:

Locales: [de-DE, en-US, …]

  1. Flutter sucht die erste Übereinstimmung in:

supportedLocales

  1. S.delegate.load(locale) wird aufgerufen
  2. S.of(context) greift automatisch auf diese Locale zu

📌 Das ist sauberes, idiomatisches Flutter

🧠 Grundprinzip (wichtig)

App-spezifische Spracheinstellungen werden vom Betriebssystem verwaltet,
nicht von deiner App.

Flutter liest nur, entscheidet aber nicht selbst.

🍎 iOS: Settings > App > Sprache

Voraussetzung

  • iOS 13+
  • App hat lokalisierte Ressourcen

Was Flutter dafür braucht

Nichts zusätzliches, außer:

⚠️ iOS-Fallstrick (wichtig!)

Du MUSST in Xcode die Sprachen deklarieren.

ios/Runner/Info.plist

<key>CFBundleLocalizations</key>

<array>

  <string>de</string>

  <string>en</string>

  <string>fr</string>

</array>

 

📌 Das ist der entscheidende Punkt, den viele vergessen.

🤖 Android: App-spezifische Sprache

Android 13+ (API 33)

  • Android zeigt automatisch:

Einstellungen

 → Apps

   → Punktspiel

     → Sprache

Flutter-Seite

Ebenfalls nichts weiter nötig, solange:

supportedLocales: S.delegate.supportedLocales,

gesetzt ist.

⚠️ Android-Fallstrick

In android/app/src/main/AndroidManifest.xml:

<application

  android:localeConfig="@xml/locales_config"

  ...>

android/app/src/main/res/xml/locales_config.xml

<?xml version="1.0" encoding="utf-8"?>

<locale-config xmlns:android="http://schemas.android.com/apk/res/android">

  <locale android:name="de"/>

  <locale android:name="en"/>

  <locale android:name="fr"/>

</locale-config>

 

📌 Ohne das zeigt Android die Sprachwahl nicht an.
